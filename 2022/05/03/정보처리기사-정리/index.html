정보처리기사 정리

Author: Mihyun Lee Date: May 3, 2022  22:25:29

화이트박스 테스트

  • 프로그램 원시 코드의 논리적인 구조를 커버하도록 테스트 케이스를 설계
  • 코드의 제어 구조 설계 절차에 초점을 맞춰 테스트 케이스를 설계하며, 주로 테스트 과정 초기에 적용
  • 제품의 내부 요소들이 명세서에 따라 수행되고 충분히 실행되는가를 보장하기 위한 검사

화이트박스 테스트의 제어 구조 검사

  • 조건 검사 : 소스코드의 논리적 조건을 테스트
  • 루프 검사 : 소스 코드의 반복 구조를 중점적으로 테스트
  • 데이터 흐름 검사 : 소스 코드의 변수 정의, 사용을 중점적으로 테스트

베타 테스트

다수의 사용자에게 제한되지 않은 환경에서 프로그램을 사용하게 하고 오류가 발견되면 개발자에게 통보하는 방식의 테스트 방법

알파 테스트

개발자의 장소에서 사용자가 개발자 앞에서 행하는 테스트 기법이며 오류와 사용상의 문제점을 사용자와 개발자가 함께 확인하면서 기록한다.

정보보호의 목표 5가지

기밀성(Confidentiality), 무결성(Integrity), 가용성 (Availability), 인증성 (Authenticity), 책임성(Accountability)

레지스터

  • CPU 내에서 발생 및 사용되는 데이터를 일시적으로 저장하는 저장장치
  • 가격대비 용량이 가장 작고 기억장치 중 속도가 가장 빠르다
  • 플립플롭과 래치로 구성된다.

레지스터의 종류

  • 제어장치의 레지스터 : CU(Control Unit), 산술 및 논리 연산 등의 연산 기능을 수행할 때 사용되는 레지스터
  • 연산장치의 레지스터 : ALU(Arithmetic Logic Unit), 명령어 수행에 필요한 데이터를 저장하는 레지스터

정규형

  • 제 1 정규형
    • 원자성을 가지지 못하는 반복 그룹 속성이 제거된 릴레이션 스키마
    • 릴레이션의 모든 속성 및 도메인이 원자성을 가진다면 제 1정규형에 해당한다.
  • 제 2 정규형
    • 부분 함수 종속을 제거한 릴레이션 스키마
    • 기본 키가 아닌 모든 속성들이 기본키에 완전함수 종속이면 제 2 정규형에 속한다.
    • 완전 함수 종속 관계를 만들기 위해 릴레이션을 기본 키 중심으로 분해한다.
  • 제 3 정규형
    • 이행적 함수 종속을 제거한 릴레이션 스키마
    • 속성 A가 속성 B를 결정하고, 속성 B가 속성 C 를 결정하면, 속성 A가 속성 C 를 결정
    • 종속자 C의 결정자가 B외에도 A 가 추가로 존재하는 종속 관계
    • 결정자이자 종속자에 해당하는 속성을 기준으로 릴레이션을 분할한다.
  • 보이스코드 정규형 (BCNF)
    • 결정자이면서 후보키가 아닌 함수 종속을 제거한 릴레이션 스키마
    • 모든 결정자가 후보키이면 BCNF에 속한다.
    • 해당되는 경우가 거의 없어 실제로 고려되지 않는 정규형이다.
  • 제 4 정규형
    • 다치 종속을 제거한 릴레이션 스키마
    • 둘 이상의 속성을 가진 결정자에 의해 정해지는 속성
  • 제 5 정규형
    • 후보키를 통하지 않은 조인 종속을 제거한 릴레이션 스키마
    • 릴레이션에 존재하는 모든 조인 종속성이 릴레이션의 후보키를 통해서만 성립된다.

서브넷 마스크

  • 배정받은 하나의 네트워크 주소를 다시 여러개의 작은 하위 네트워크로 나누어 사용하는 방식

FLSM(Fixed Length Subnet Mask)

  • 동일한 크기로 서브넷을 나누는 방식

VLSM(Variable Length Subnet Mask)

  • 다양한 크기로 서브넷을 나누는 방식

DLT (Distributed Ledger Technology, 분산 원장 기술)

  • 중앙 관리자나 중앙 데이터 저장소가 존재하지 않고 P2P망 내의 참여자들에게 모든 거래 목록이 분산 저장되어 거래가 발생할 때마다 지속적으로 갱신되는 디지털 원장을 의미한다.

CIDR (Classless Inter-Domain Routing)

  • IPv4의 주소 부족 문제 해결 방안 중 하나로, 기존의 IPv4 의 클래스 체계를 무시하고 네트워크 주소와 호스트 주소를 임의로 구분하여 사용하는 방식

그레이웨어 (Grayware)

  • 소프트웨어를 제공하는 입장에서는 악의성이 없더라도 사용자 입장에서는 유용하거나 악의적이라고 판단될 수 있는 애드웨어, 공유웨어, 스파이웨어 등의 총칭이다.

디지털 저작권 관리(DRM)

  • 인터넷이나 기타 디지털 매체를 통해 유통되는 데이터의 저작권 보호를 위해 데이터의 안전한 배포를 활성화하거나 불법 배포를 방지하기 위한 시스템이다.
